OpenSeadragon.TileSource(width, height, tileSize, tileOverlap, minLevel, maxLevel)
The TileSource contains the most basic implementation required to create a smooth transition between layer in an image pyramid. It has only a single key interface that must be implemented to complete it key functionality: 'getTileUrl'. It also has several optional interfaces that can be implemented if a new TileSource wishes to support configuration via a simple object or array ('configure') and if the tile source supports or requires configuration via retreival of a document on the network ala AJAX or JSONP, ('getImageInfo').
By default the image pyramid is split into N layers where the images longest side in M (in pixels), where N is the smallest integer which satisfies 2^(N+1) >= M.
Parameters:
{Number|Object|Array|String} width
If more than a single argument is supplied, the traditional use of positional parameters is supplied and width is expected to be the width source image at it's max resolution in pixels. If a single argument is supplied and it is an Object or Array, the construction is assumed to occur through the extending classes implementation of 'configure'. Finally if only a single argument is supplied and it is a String, the extending class is expected to implement 'getImageInfo' and 'configure'.
{Number} height
Width of the source image at max resolution in pixels.
{Number} tileSize
The size of the tiles to assumed to make up each pyramid layer in pixels. Tile size determines the point at which the image pyramid must be divided into a matrix of smaller images.
{Number} tileOverlap
The number of pixels each tile is expected to overlap touching tiles.
{Number} minLevel
The minimum level to attempt to load.
{Number} maxLevel
The maximum level to attempt to load.
